Getting Started

Catalyst Networks is an open-source platform for managing Nebula mesh VPN networks. This guide walks you through initial setup.

Prerequisites

  • Docker and Docker Compose installed
  • Git
  • A terminal / command line

1. Clone and Configure

git clone https://github.com/your-org/catalyst-networks-open-source.git
cd catalyst-networks-open-source
cp .env.example .env

Generate the required secrets and add them to your .env file:

# Generate a Django secret key
python3 -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"

# Generate a registration token
python3 -c "import secrets; print(secrets.token_urlsafe(32))"

2. Start the Services

docker compose up --build -d

This starts the web application, PostgreSQL, Redis, Celery worker, and Celery beat scheduler.

3. Initialize the Database

docker compose exec web python manage.py migrate
docker compose exec web python manage.py createsuperuser

4. First-Time Setup

Once logged in, follow these steps to get your first network running:

  1. Create an Organization — Go to Organizations and create one. This is your tenant boundary.
  2. Add a Network Range — In your organization, add a CIDR range (e.g., 10.42.0.0/24). This is the IP pool for your Nebula nodes.
  3. Create a Certificate Authority — Go to Certificates and create a CA. This generates the Nebula CA keypair used to sign node certificates.
  4. Add Your First Node — Create a node (start with a lighthouse). The platform auto-assigns an IP and generates a signed certificate.
  5. Download the Config — Download the node's configuration bundle (cert, key, and YAML config) and deploy it to your server.
