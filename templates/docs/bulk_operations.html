Bulk Operations

Manage nodes at scale with CSV import/export, bulk deletion, and batch certificate renewal.

CSV Export

Export all nodes in an organization as a CSV file. Go to Nodes → Export CSV (requires Admin or Owner role).

The export includes these columns:

ColumnDescription
nameNode hostname
nebula_ipAssigned Nebula IP address
is_lighthouseTrue/False
public_ipPublic IP (lighthouses only)
fqdnFQDN (lighthouses only)
external_portUDP port (default 4242)
assigned_user_emailEmail of assigned user
security_groupsComma-separated group names
cert_expirationCertificate expiry date (ISO 8601)
last_checkinLast heartbeat (ISO 8601)
created_atCreation date (ISO 8601)

CSV Import

Bulk-create nodes by uploading a CSV file. Go to Nodes → Import CSV.

Required Columns

Only the name column is required. IPs are auto-assigned and certificates are auto-generated.

Optional Columns

  • is_lighthouse — true, false, 1, 0, or yes/no
  • public_ip — Required if is_lighthouse is true (or provide fqdn)
  • fqdn — Fully qualified domain name for lighthouses
  • external_port — UDP port, defaults to 4242

Example CSV

name,is_lighthouse,public_ip,fqdn,external_port
web-01,false,,,
web-02,false,,,
db-primary,false,,,
db-replica,false,,,
lighthouse-us,true,203.0.113.1,lh-us.example.com,4242
lighthouse-eu,true,198.51.100.5,lh-eu.example.com,4242

Import Behavior

  • Nodes with duplicate names (already existing in the org) are skipped
  • Each node gets an auto-assigned IP from the organization's network range
  • Certificates are generated and signed by the latest CA
  • Lighthouse nodes without a public_ip or fqdn are skipped with an error
  • A summary of created nodes and any errors is shown after import

Bulk Delete

Go to Nodes → Bulk Delete to remove multiple nodes at once. Select nodes using the checkboxes and confirm deletion.

Warning: Deletion is permanent. The node, its certificate, and private key are all removed. This cannot be undone.

Bulk Certificate Renewal

Go to Nodes → Bulk Renew to renew certificates for multiple nodes. Use the expiration window filter to find nodes with certificates expiring soon:

  • Filter by window: 7, 14, 30, 60, 90 days, or all
  • Select all or individual nodes
  • Click "Renew Selected Certificates" to regenerate

Via Management Command

# Preview what would be renewed
docker compose exec web python manage.py renew_certificates --days 30 --dry-run

# Renew all expiring certificates
docker compose exec web python manage.py renew_certificates --days 30

# Force renew entire organization
docker compose exec web python manage.py renew_certificates --organization-id 1 --force
