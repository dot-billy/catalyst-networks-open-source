Troubleshooting

Common issues and how to resolve them.

"No certificate authority exists"

Cause: You're trying to create a node, but no CA has been created for this organization.

Fix: Go to Certificates → Create Certificate Authority in your organization. You need the nebula-cert binary available in your Docker container (it's included in the default Dockerfile).

"No network range configured"

Cause: The organization doesn't have a CIDR range for IP allocation.

Fix: Go to your organization's detail page and add a network range (e.g., 10.42.0.0/24).

"DJANGO_SECRET_KEY environment variable is not set"

Cause: The .env file is missing or doesn't have a DJANGO_SECRET_KEY value.

Fix:

cp .env.example .env
# Generate and set DJANGO_SECRET_KEY:
python3 -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"

Certificate generation fails

Cause: The nebula-cert binary is not found or not executable.

Fix: Verify nebula-cert is installed in the container:

docker compose exec web which nebula-cert
docker compose exec web nebula-cert --version

If not present, check your Dockerfile includes the Nebula binary download step.

Database connection errors

Cause: PostgreSQL is not ready or credentials are wrong.

Fix:

# Check if the database container is running
docker compose ps db

# Check database logs
docker compose logs db

# Verify your .env has the correct values:
# POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_HOST=db

Celery tasks not running

Cause: Celery worker or beat is not started, or Redis is unavailable.

Fix:

# Check all services are running
docker compose ps

# Check Celery worker logs
docker compose logs celery

# Check Celery beat logs
docker compose logs celery-beat

# Check Redis connectivity
docker compose exec web python -c "import redis; r = redis.Redis(host='redis'); print(r.ping())"

Nodes can't find each other

Cause: Lighthouse is unreachable, or node configs are stale.

Fix:

  • Verify the lighthouse's public IP/FQDN is correct and the UDP port (default 4242) is open
  • Re-download the node config from the web UI — it includes the latest lighthouse addresses
  • Check that the CA certificate on each node matches the CA that signed its certificate
  • Run nebula -config config.yaml -test on the node to validate the config

Rate limit exceeded on API registration

Cause: Too many registration requests from the same IP or organization.

Fix: Wait for the rate limit window to reset (1 hour). Limits are: 5 per IP per hour, 10 per organization per hour. For large deployments, space out registrations or use the CSV import feature instead.

Running the healthcheck

The built-in healthcheck script verifies Django can start, connect to the database, and import the WSGI application:

docker compose exec web python healthcheck.py

The health endpoint is also available at /health/ for monitoring tools.

Getting Help

If your issue isn't covered here, open an issue on the GitHub repository with:

  • Steps to reproduce the problem
  • Relevant error messages or logs
  • Your environment (OS, Docker version)
