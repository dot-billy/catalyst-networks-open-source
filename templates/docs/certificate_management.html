Certificate Management

Catalyst Networks uses Nebula's built-in PKI to manage Certificate Authorities and node certificates. Every node gets a signed certificate that proves its identity on the mesh.

Certificate Authority (CA)

Each organization has one or more CAs. The CA keypair is used to sign all node certificates in that organization. When you create a CA, the platform runs nebula-cert ca to generate the keypair.

Security: The CA private key is the root of trust for your entire network. Protect it accordingly. In production, consider using encrypted storage for the CERT_STORAGE_ROOT volume.

To create a CA:

  1. Navigate to your organization
  2. Go to Certificates
  3. Click Create Certificate Authority
  4. Provide a name (e.g., "Production CA 2026")

Node Certificates

When a node is created (via the web UI, API, or CSV import), a certificate is automatically generated and signed by the organization's latest CA. The certificate includes:

  • The node's name
  • Its assigned Nebula IP (with /24 subnet)
  • Security group memberships (as Nebula groups)
  • An expiration date

Certificate Renewal

Certificates can be renewed in three ways:

Manual (Single Node)

From the node detail page, click Renew Certificate. This regenerates the cert with the same parameters.

Bulk Renewal (Web UI)

Go to Nodes → Bulk Renew. Filter by expiration window (7/14/30/60/90 days), select nodes, and renew all at once. See Bulk Operations for details.

Management Command

# Renew all certs expiring within 14 days
docker compose exec web python manage.py renew_certificates

# Renew for a specific organization
docker compose exec web python manage.py renew_certificates --organization-id 1

# Renew a specific node
docker compose exec web python manage.py renew_certificates --node-id 42

# Dry run (see what would be renewed)
docker compose exec web python manage.py renew_certificates --dry-run

# Force renew regardless of expiration
docker compose exec web python manage.py renew_certificates --organization-id 1 --force

Expiration Monitoring

Catalyst Networks monitors certificate expiration automatically:

  • Dashboard warnings — Certificates expiring within 30 days appear as alerts on the dashboard
  • Celery Beat — A scheduled task checks for expiring certificates and sends webhook notifications
  • Webhook events — cert.expiring events are fired for certificates approaching expiration

Configuration in .env:

CERTIFICATE_RENEWAL_WINDOW_DAYS=14     # Auto-renew window
CERTIFICATE_RENEWAL_BATCH_SIZE=10      # Batch size for processing
CERTIFICATE_EXPIRY_NOTIFICATION_DAYS=30  # Webhook alert window

Mobile Device Enrollment

For mobile devices (phones, tablets), Catalyst Networks supports QR code enrollment:

  1. Create a mobile node and assign it to a user
  2. A QR code is automatically generated
  3. The user scans the QR code with the Nebula mobile app
  4. The enrollment endpoint provides the signed certificate bundle

QR enrollment tokens expire after 7 days by default.
