API Reference

Catalyst Networks provides a REST API for node registration and management. Interactive documentation is available at /api/docs/ (Swagger) and /api/redoc/ (ReDoc).

Authentication

The API supports two authentication methods:

JWT Tokens (User Authentication)

Obtain a token pair by posting credentials:

POST /api/token/
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "your-password"
}

# Response:
{
  "access": "eyJ0eXAiOiJKV1Qi...",
  "refresh": "eyJ0eXAiOiJKV1Qi..."
}

Use the access token in subsequent requests:

Authorization: Bearer eyJ0eXAiOiJKV1Qi...

Registration Tokens (Fleet Deployment)

For automated deployments, use registration tokens created in the web UI. These are time-limited and can restrict the number of uses and node types.

Node Registration

Register a new node and receive its certificate bundle.

Authenticated Registration

POST /api/org/{slug}/nodes/register/
Authorization: Bearer {jwt_token}
Content-Type: application/json

{
  "node_name": "web-server-01",
  "is_lighthouse": false
}

Token-Based Registration

POST /api/nodes/register/
Content-Type: application/json

{
  "organization_slug": "my-org",
  "node_name": "edge-node-05",
  "registration_token": "your-registration-token",
  "is_lighthouse": false
}

Lighthouse Registration

POST /api/org/{slug}/nodes/register/
Authorization: Bearer {jwt_token}
Content-Type: application/json

{
  "node_name": "lighthouse-east",
  "is_lighthouse": true,
  "public_ip": "203.0.113.1",
  "fqdn": "lh-east.example.com",
  "external_port": 4242
}

Response

HTTP 200 OK
Content-Type: application/json

{
  "node_id": 42,
  "node_name": "web-server-01",
  "nebula_ip": "10.42.0.5",
  "is_lighthouse": false,
  "api_token": "a1b2c3d4...",
  "certificate": "-----BEGIN NEBULA CERTIFICATE-----\n...",
  "key": "-----BEGIN NEBULA X25519 PRIVATE KEY-----\n...",
  "ca_certificate": "-----BEGIN NEBULA CERTIFICATE-----\n...",
  "config_yaml": "pki:\n  ca: /etc/nebula/ca.crt\n  ...",
  "expiration": "2027-03-27T00:00:00Z"
}

Node Management Endpoints

MethodEndpointDescription
GET/api/org/{slug}/nodes/List all nodes in an organization
POST/api/org/{slug}/nodes/register/Register a new node
GET/api/org/{slug}/nodes/{id}/Get node details
PATCH/api/org/{slug}/nodes/{id}/Update a node
POST/api/org/{slug}/nodes/{id}/renew_cert/Renew node certificate
POST/api/org/{slug}/nodes/{id}/checkin/Node heartbeat check-in
GET/api/org/{slug}/nodes/{id}/security_groups/Get node's security groups

Error Responses

All error responses follow this format:

{
  "error": "A human-readable error message"
}
StatusMeaning
400Bad request (missing fields, invalid data)
401Unauthorized (invalid or missing token)
403Forbidden (insufficient permissions)
404Organization or node not found
429Rate limit exceeded
500Internal server error
