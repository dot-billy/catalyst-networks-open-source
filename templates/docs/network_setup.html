Network Setup

Configure CIDR ranges and understand how Catalyst Networks allocates IP addresses across your Nebula mesh.

Network Ranges

Each organization has one or more network ranges defined as CIDR blocks. When a node is created, Catalyst Networks automatically assigns the next available IP from the range.

Only private (RFC 1918) and CGNAT ranges are allowed:

RangeCIDRUsable Hosts
Class A Private10.0.0.0/8~16.7 million
Class B Private172.16.0.0/12~1 million
Class C Private192.168.0.0/16~65,000
CGNAT100.64.0.0/10~4 million

Choosing a Range

Pick a range that doesn't conflict with existing infrastructure on your nodes' host networks. Common choices:

  • 10.42.0.0/24 — 254 hosts, good for small teams
  • 10.42.0.0/16 — ~65,000 hosts, room to grow
  • 100.64.0.0/24 — CGNAT range, avoids conflicts with most LANs

IP Allocation

IPs are assigned sequentially from the first available address in the range. The allocation algorithm:

  1. Gets all network ranges for the organization
  2. Enumerates host addresses in the first range
  3. Skips any IP already assigned to an existing node
  4. Returns the first unused IP
Note: You can also manually specify an IP when creating a node via the API. The IP must fall within one of the organization's network ranges.

Lighthouses

Lighthouse nodes are the rendezvous points in a Nebula network. They help other nodes find each other. Every organization needs at least one lighthouse before regular nodes can be added.

Lighthouse requirements:

  • A public IP address or FQDN reachable by all other nodes
  • A UDP port (default: 4242) open in the host firewall
  • Stable uptime — all nodes depend on the lighthouse for discovery

Adding a Network Range

  1. Navigate to your organization's detail page
  2. In the Network Ranges section, enter a CIDR block
  3. Add an optional description (e.g., "Production overlay network")
  4. Save — the range is now available for node IP allocation
